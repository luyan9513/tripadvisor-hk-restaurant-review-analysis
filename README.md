# TripAdvisor Hong Kong Restaurant Review Analysis

This repository contains a reproducible pipeline for collecting, cleaning, analyzing, and modeling TripAdvisor restaurant reviews for Hong Kong restaurants. It includes the raw data exports, feature-engineering scripts, exploratory analysis, sentiment and emotion extraction, and baseline classification experiments.

## Project Scope

- Crawl and normalize TripAdvisor restaurant and review data.
- Prepare analysis-ready review datasets with text, metadata, and temporal features.
- Run exploratory data analysis on ratings, trip types, price range, and review volume.
- Extract sentiment and emotion features using lexicon-based methods and VADER.
- Benchmark baseline models for five-star review prediction.
- Generate figures and summary tables for reporting and presentation.

## Repository Layout

- `data/raw/`: source CSV files used by the pipeline.
- `scripts/`: data preparation, EDA, feature extraction, modeling, and visualization scripts.
- `resources/`: sentiment and emotion lexicons used by the feature-extraction pipeline.
- `outputs/`: derived datasets, plots, and markdown summaries generated by the scripts.
- `tripadvisor_hk_restaurants_reviews_crawl.py`: crawler client for refreshing the raw TripAdvisor exports.
- `report.pdf` and `report.docx`: written project report.
- `Quartet-V2.pptx`: project presentation deck.

## Requirements

- Python 3.11 or newer
- Packages listed in `requirements.txt`
- NLTK corpora:

```bash
python -m nltk.downloader opinion_lexicon vader_lexicon
```

## Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Reproducible Workflow

Run the scripts in this order:

1. Prepare the modeling datasets:

```bash
python scripts/prepare_modeling_data.py
```

2. Generate exploratory analysis outputs:

```bash
python scripts/run_eda.py
```

3. Add sentiment and emotion features:

```bash
python scripts/extract_sentiment_emotion_features.py
```

4. Train baseline models:

```bash
python scripts/train_baseline_models.py
```

5. Produce the sentiment/emotion visualization summary:

```bash
python scripts/visualize_sentiment_emotion.py
```

## Optional Data Refresh

If you have a TripAdvisor Scraper API key, you can refresh the raw CSV exports:

```bash
export API_KEY=your_api_key
python tripadvisor_hk_restaurants_reviews_crawl.py
```

The crawler writes the raw outputs to `data/raw/`.

## Notes

- The repository keeps raw source data under `data/raw/` so the workflow is self-contained.
- The crawler depends on a working TripAdvisor scraper API key.
- The modeling pipeline assumes English review text and filters out noisy or mixed-language samples.
- The `outputs/` directory is ignored by git because it only contains generated artifacts.
